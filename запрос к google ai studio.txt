Фронт - Chrome расширение, которое позволяет пользователю общаться

Бэк - Langgraph, сначала он должен работать локально, потом, когда он заработает локально, мы его задеплоим на бесплатном тарифе Render

Что должен уметь делать агент:

1) Это расширение выглядит как чат окно в Chrome, этот агент должен считывать содержимое страницы и использовать содержимое страницы при ответе на вопрос пользователя. То есть пользователь может спросить суммировать страницу или что-то пояснить на странице и агент должен использовать содержимое страницы при ответе. Общение с агентом идет в режиме чата. История чат сессий не нужна, но подумай как должна быть реализована память такого агента, чтобы он помнил сообщения текущей сессии.

2) Агент должен использовать Gemini Flash 2.5

3) Когда агент делает что либо (думает, использует инструменты, готовит ответ) он должен показывать что он сейчас делает, на какой он стадии.

1. Интерфейс агента должен быть минималистичным но аккуратным и стильным, используй Shadcn для всех эелементов интерфейса
2. У агента есть инструмент:

Использование API EXA для ресерча чего либо. Например если я прошу найти скидочный код для товара, он должен взять информацию со страницы которую я сейчас смотрю и поискать скидочные коды на этот товар. Find webpages using Exa's embeddings-based or Google-style keyword search.

# Search

> The search endpoint lets you intelligently search the web and extract contents from the results.

By default, it automatically chooses between traditional keyword search and Exa's embeddings-based model, to find the most relevant results for your query.

# Answer

> Get an LLM answer to a question informed by Exa search results. `/answer` performs an Exa search and uses an LLM to generate either:
1. A direct answer for specific queries. (i.e. "What is the capital of France?" would return "Paris")
2. A detailed summary with citations for open-ended queries (i.e. "What is the state of ai in healthcare?" would return a summary with citations to relevant sources)

The response includes both the generated answer and the sources used to create it. The endpoint also supports streaming (as `stream=True`), which will return tokens as they are generated.

Alternatively, you can use the OpenAI compatible [chat completions interface](https://docs.exa.ai/reference/chat-completions#answer).
